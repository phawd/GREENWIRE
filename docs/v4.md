# GREENWIRE v2.0 - Intelligent Card System Architecture

## Overview

**Intelligent Card System v2.0** is a comprehensive EMV/JavaCard security research platform with:

- **AI-Enhanced Testing**: Machine learning-driven test selection from library of 56 EMVCo/PCI tests
- **Production Mode**: Generate real EMVCo-compliant test cards for internal security testing
- **HSM/ATM Integration**: Cryptographic operations (PIN, CVV, MAC, ARQC/ARPC) with bidirectional learning
- **Continuous Learning**: Cards ↔ HSM/ATM ↔ Merchants share intelligence across ecosystem
- **Vulnerability Research**: Advanced fuzzing with AI-powered attack recommendation

### 1. Intelligent Card System (intelligent_card_system.py)

**Main Orchestrator** - Coordinates all components for complete card intelligence workflow.

#### Key Features

- **Production Mode**: Generate real EMVCo-compliant test cards
- **AI-Enhanced Testing**: 20-30 tests selected from library of 56
- **HSM/ATM Integration**: Cryptographic operations with bidirectional learning
- **Continuous Learning**: All components share intelligence

#### Class: IntelligentCardSystem

```python
class IntelligentCardSystem:

                                 cardholder_name: str, metadata: Dict) -> Dict

    def run_merchant_tests(card_interface, card_id: str, merchant_id: str,

    def extract_on_card_data(card_interface) -> Dict

    def close()
```

### 3. Merchant Test Library (merchant_test_library.py)

**EMVCo/PCI DSS Test Repository** - 56 production-grade tests across 8 categories.

#### Test Categories (8)

| Category | Description | Test Count |
|----------|-------------|------------|
| `APPLICATION_SELECTION` | AID selection, file structure | 8 tests |
| `CRYPTOGRAM_GENERATION` | ARQC/TC/AAC generation | 7 tests |
| `AUTHENTICATION` | DDA, CDA, SDA methods | 8 tests |
| `CARDHOLDER_VERIFICATION` | PIN, signature, biometric | 6 tests |
| `TRANSACTION_FLOW` | Online/offline authorization | 9 tests |
| `SECURITY_FEATURES` | Anti-cloning, anti-skimming | 8 tests |
| `ERROR_HANDLING` | Fault injection, edge cases | 6 tests |
| `PERFORMANCE` | Timing, throughput, concurrency | 4 tests |

#### Test Severity Levels (5)

| Severity | Score | Description |
|----------|-------|-------------|
| `CRITICAL` | 1.0 | Card cloning, key extraction |
| `HIGH` | 0.8 | Authorization bypass, PIN bypass |
| `MEDIUM` | 0.6 | Data leakage, timing attacks |
| `LOW` | 0.4 | Compliance violations |
| `INFORMATIONAL` | 0.2 | Best practice violations |

### 4. HSM/ATM Integration (hsm_atm_integration.py)

**Cryptographic Operations & Ecosystem Intelligence** - HSM operations with bidirectional learning.

#### Key Features

- **HSM Operations**: PIN, CVV, MAC, ARQC/ARPC validation
- **ATM Simulation**: Withdrawal, balance, authentication
- **Bidirectional Learning**:
  - **Cards → HSM/ATM**: Vulnerability reports, attack patterns
  - **HSM/ATM → Cards**: Test recommendations, risk assessments
- **Ecosystem Database**: Shared knowledge across all cards and merchants

#### Class: HSMATMIntegration

```python
class HSMATMIntegration:
    def __init__(self)
    
    # HSM Operations
    def generate_pin_key(card_id: str) -> str
    def translate_pin(from_format: str, to_format: str, 
                     encrypted_pin: str, card_id: str) -> str
    def verify_pin(card_id: str, encrypted_pin: str) -> bool
    def generate_cvv(card_number: str, expiry_date: str, 
                     service_code: str) -> str
    def verify_cvv(card_number: str, expiry_date: str, 
                   service_code: str, cvv: str) -> bool
    def generate_mac(card_id: str, data: bytes) -> str
    def validate_arqc(card_id: str, arqc: str, 
                     transaction_data: Dict) -> bool
    def generate_arpc(card_id: str, arqc: str, 
                     transaction_data: Dict) -> str
    
    # ATM Operations
    def atm_withdraw(card_id: str, amount: float, currency: str) -> Dict
    def atm_balance_inquiry(card_id: str) -> Dict
    def atm_card_authentication(card_id: str, method: str) -> bool
    def atm_pin_entry(card_id: str, pin: str) -> bool
    def atm_transaction_log(card_id: str, transaction_type: str, 
                           amount: float) -> Dict
    
    # Bidirectional Learning
    def receive_card_report(card_id: str, merchant_id: str, 
                           report_type: str, report_data: Dict)
    def generate_test_recommendations(merchant_id: str, 
                                     card_id: str) -> List[Dict]
    def get_merchant_intelligence(merchant_id: str) -> Dict
```

## Security Considerations

### Production Mode

⚠️ **Production mode generates REAL cryptographic keys and EMVCo-compliant test cards.**

**Best Practices**:

1. **Access Control**: Restrict production mode to authorized personnel only
2. **Audit Trail**: All production card generation logged with timestamp, purpose, and operator
3. **Secure Storage**: Store production cards in secure physical vault
4. **Usage Tracking**: Log all test card usage in test environment
5. **Destruction Policy**: Destroy cards after testing complete (shred & degauss)
6. **Never Use for Real Transactions**: Production cards are for INTERNAL TESTING ONLY

**Compliance**:

- **PCI DSS**: Production cards comply with PCI PTS POI requirements
- **EMVCo**: Cards generated per EMVCo Book 2 & 3 specifications
- **Internal Security**: Audit trail for security team review

### Data Protection

**Sensitive Data Storage**:

- **Card Numbers**: Masked in logs (e.g., `4000 **** **** 5678`)
- **CVV**: Stored in card data file (encrypted in production deployment)
- **Keys**: Generated via HSM, never stored in plaintext
- **Audit Logs**: Retained for 1 year minimum per compliance requirements

**Database Encryption** (Production Deployment):

- SQLite databases should be encrypted at rest (SQLCipher)
- HSM keys should be hardware-backed (not software-simulated)
- Card data files should be encrypted with AES-256

### Example 3: Continuous Learning Integration

```python
from modules.intelligent_card_system import IntelligentCardSystem
from modules.ai_test_generator import AITestGenerator
from modules.hsm_atm_integration import HSMATMIntegration

# Initialize components
ics = IntelligentCardSystem(production_mode=False)
test_generator = AITestGenerator()
hsm_atm = HSMATMIntegration()

# Simulate multiple test cycles (continuous learning)
for cycle in range(10):
    print(f"\n=== Learning Cycle {cycle + 1}/10 ===")
    
    # Step 1: Get test recommendations from HSM/ATM (ecosystem intelligence)
    recommendations = hsm_atm.generate_test_recommendations(
        merchant_id="MERCHANT_XYZ",
        card_id=f"CARD-{cycle:03d}"
    )
    print(f"HSM/ATM recommended {len(recommendations)} priority tests")
    
    # Step 2: Generate AI test mix
    test_mix = test_generator.generate_test_mix(
        merchant_id="MERCHANT_XYZ",
        card_capabilities={"contact": True, "contactless": True, "dda": True},
        test_count=20
    )
    print(f"AI selected {len(test_mix)} tests from library")
    
    # Step 3: Execute tests (simulated)
    for test in test_mix:
        # Simulate test execution
        success = random.random() < 0.12  # 12% vulnerability detection rate
        severity = random.uniform(0.3, 0.9) if success else 0.0
        
        # Record result (continuous learning)
        test_generator.record_test_result(
            test_id=test["test_id"],
            merchant_id="MERCHANT_XYZ",
            success=success,
            severity=severity,
            vulnerability_found=test.get("vulnerability_check") if success else None,
            execution_time_ms=random.randint(50, 200)
        )
        
        # If vulnerability found, report to HSM/ATM (Cards → HSM/ATM)
        if success:
            hsm_atm.receive_card_report(
                card_id=f"CARD-{cycle:03d}",
                merchant_id="MERCHANT_XYZ",
                report_type="vulnerability",
                report_data={
                    "vulnerability_type": test["test_id"],
                    "severity": severity,
                    "attack_vector": test.get("name"),
                    "exploit_success_rate": 1.0
                }
            )
    
    # Step 4: View learning progress
    merchant_intel = test_generator.get_merchant_intelligence("MERCHANT_XYZ")
    print(f"Merchant Risk Score: {merchant_intel['risk_score']:.2f}")
    print(f"Known Vulnerabilities: {len(merchant_intel['vulnerabilities'])}")

# Step 5: Retrain ML model with all collected data
print("\n[AI] Retraining ML model with new intelligence...")
test_generator.retrain_model()
print("✅ ML model updated")

ics.close()
```
